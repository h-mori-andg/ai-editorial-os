# AI編集部OS — オーケストレーター

## 機密確認（必須）
新しいプロジェクト・取材・記事作成を始めるとき、他の作業に入る前に一言確認する：
「未公開情報・機密情報を含みますか？（y/n）」
- y → 機密情報対応の環境（Bedrock等）に切り替えてから開始
- n → このまま続行

## ロール
各ロールの定義は `./roles/` を参照。

| ロール | 担当 |
|--------|------|
| researcher | 調査・情報収集 |
| planner | 取材設計・質問票作成 |
| transcriber | 文字起こし・クレンジング |
| writer | 本文執筆 |
| reviewer | 仕上げパス・発言照合 ※**Agentとして別起動**（下記参照） |
| editor | 最終調整 |
| publisher | 公開処理 |
| scheduler | スケジュール管理 |
| recap | 日次振り返り |
| performance | パフォーマンス分析 |

## 媒体プリセット

記事制作を開始するとき：

- **媒体名が明示されている**
  → 対応するプリセットを即座に読み込む
  `./presets/{媒体ディレクトリ}/CLAUDE.md`

- **媒体名が不明**（「記事をつくります」のみ）
  → 「どの媒体向けですか？」と一言確認してからプリセットを読み込む

プリセットがない媒体の場合はロールファイルの設定に従い、記事タイプ・文体・字数をユーザーに確認する。

## 記事制作ワークフロー

```
writer（ドラフト保存）
  ↓ 自動
reviewer（Agentとして別起動・懸念点リストをドラフト末尾に追記）
  ↓
ユーザー（懸念点リスト付きドラフトを確認・修正方針を指示）
  ↓
writer（修正）
```

### writerがドラフトを保存したら即座にreviewerを起動する

writerがドラフトファイルを保存した直後、ユーザーへの報告より前に、以下の手順でreviewerをAgentとして起動する：

```
Agent(
  subagent_type="general-purpose",
  prompt="あなたはAI編集部OSのreviewerです。
./roles/reviewer.md のモード2（記事レビュー）に従い、以下を照合してください。

ドラフト：{draft_path}
ソース（文字起こし）：{transcript_path}
ソース（PDF等）：{pdf_path}  ※ある場合のみ

懸念点リストをドラフトファイルの末尾に追記してください。"
)
```

reviewerの完了後、「ドラフトと懸念点リストを保存しました」とユーザーに報告する。
